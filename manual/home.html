This git repository is used for a live pattern recognition of coin sound signals.
The coins are thrown into a bowl and the signal is recorded by a microphone array.
The plan shows how to place the microphones and the bowl in 3 variations with 2 bowls. The coins are thrown through the slot above the bowl.

How to perform live pattern recognition Keyword Index Function Index
For prepairing a live experiment follow the steps below, if you have already trained models you can start at step 3.
  1. Execute the FEA.xtp script in the directory of the actual experiment with following parameters:
    • $1 ana
    • $2 C:\users\[USERNAME]\workspace\uasr-data\coins\[EXPERIMENTNAME]\info\HMM-trn.cfg
    If you are using the windows command prompt, be sure to specify the absolute file path for FEA.xtp.
    Alternatively you can use the preset run configurations in eclipse,
    only be sure to take the correct one for the experiment.
  2. Execute the HMM.trn script in the same directory with following parameters:
    • $1 trn
    • $2 C:\users\[USERNAME]\workspace\uasr-data\coins\[EXPERIMENTNAME]\info\HMM-trn.cfg
    The 2nd Parameter has to be equal as in the FEA.xtp script, also be sure to use absolute file paths.
  3. Execute the REC_PACKDATA.trn script in the same directory with following parameters:
    • $1 rec
    • $2 info/REC_PACKDATA.cfg
    • $3 -v2
    The Path ($2) of the config-file is relative to the working directory and has to define the working model.
  4. Start Scarlett MixControl for feedback setup
    1. turn the last volume knob (labeled with a phones icon) to a low level, e.g. "2" or "3"
      because the feedback can be dangerous for your ears if you are wearing the headphones
    2. if a connection error occurs, be sure that the power line is activated and the soundcard is running
    3. be sure that there is a feedback line
      from OUTPUT 1 at the back-side to INPUT 1 at the front
    4. be sure that the phones are plugged in the socket labeled with the phones icon
    5. open LoopRec.8i6
    6. play a sound (longer sounds recommended) in windows, e.g. with the windows media player
    7. turn the 1st knob (labeled GAIN) to optimal feedback level (should be at "8")
    8. if the 1st and the last two volume indicators move if the sound is playing
      and you do not hear a beep noise on the phones everything is fine.
    9. do not close Scarlett Mix Control, yet
  5. Start NI LabVIEW 2013 and open "Mikrofonfeld 3.1.vi" from path "C:\National Instruments Download"
    1. go to the menu "Ansicht" and activate "Werkzeugpalette"
    2. choose the hand from the new window
    3. change "Anzahl der Aufnahmen" from 10 to 1000
    4. change modus from "getriggert" to "kontinuierlich"
    5. activate 2nd blue button (the one with a triangle) so triangle turns green
    6. turn the knob labeled "Volume Ausgabe" to 100
    7. activate all 4 channels by pushing the red buttons labeled "Kanal X An/Aus"
      (X stands for the channel number) so they turn green
    8. additional if you want to record define filepaths in "Speicherpfad Summe" and "Speicherpfad Einzelaufnahme" and activate the 1st blue button so it turns red
    9. run the vi by clicking the arrow in the top left corner or select from menu "Ausführen" entry "starten"
    10. you shall now see some noise patterns in the 4 graphs on the left
    11. level "Verstärkung Summe" to "0,05" and in case of record level "Verstärkung Einzelkanäle" to "0,1", too
    12. now you can throw coins for test purposes into the bowl,
      if you see now in the right graphs some signals, too and the level is not above 1 the signal should go out
    13. change to Scarlett Mix Control (do not stop the vi) and throw again coins into the bowl,
      if the 1st and the last two volume indicator move everything is fine
    14. if you have an assistant you can check the correct signal level on the phones while he is throwing the coins
    15. now you can quit Scarlett Mix Control
    16. stop the vi by pressing the STOPP Button but do not close it
  6. Start Eclipse and open java run configuration "CoinsRecognizer"
    1. alternatively you can open package "uasr-data-coins\src\de.tucottbus.kt.uasr_data_coins"
      and run "CoinsRecognizer.java"
    2. a gui should appear and show you a green circle - that means no coin is recognized.
    3. start the vi "Mikrofonfeld 3.1"
    4. throw a coin in the bowl
    5. the green circle should have changed to a photo of a coin
    6. the questionmark indicates a refused recognition, the exclamation mark shows an accepted recognition
    7. now you can perform the live recognition as long as "Mikrofonfeld 3.1.vi" runs
  7. after the experiment
    • you should close the gui by pressing the "X" on the caption
    • you have to exit CoinsRecognizer by typing "exit" + [Enter] into the Eclipse Console
    • you should stop "Mikrofonfeld 3.1.vi"
    • you can close LabVIEW and Eclipse
    • you can close Scarlett MixControl
    • you can power off the Sound Card at the power point
